How it works

  • Client ↔ Server — each machine binds to this KMS via EPID and machine ID stored in the database.
  • Renewal — activated clients re-contact the server about every {{ policy.renewal_interval_fmt }}.
  • Activation retry — if activation fails, clients retry every {{ policy.activation_interval_fmt }}.
  • Grace — “Notifications Mode” means activation is pending; Windows runs in notification grace.
  • Changing intervals requires updating KMS container env and restarting the kms service.
